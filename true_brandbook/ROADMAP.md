# Design Terminal — Roadmap

## Phase 0 — Инфраструктура (в процессе)

- [ ] Supabase проект, схема БД, RLS политики
- [x] Next.js проект, Tailwind v4, shadcn/ui (base-nova)
- [x] next-intl (EN + RU)
- [ ] Supabase Auth, invite flow
- [ ] Railway деплой, Dockerfile с Puppeteer
- [ ] Переменные окружения

## Phase 1 — Брендбук

- [ ] Загрузка брендбука через Supabase Dashboard (MVP — руками)
- [ ] Страница просмотра брендбука для admin
- [ ] Скачивание шрифтов при сохранении брендбука
- [ ] Блокировка генерации без брендбука

## Phase 2 — Генерация

- [ ] Форма запроса (промпт + чекбоксы форматов + референс)
- [ ] API генерации с параллельными вызовами Claude
- [ ] Валидация HTML
- [ ] Puppeteer рендер JPEG превью
- [ ] Supabase Realtime — прогресс по мере готовности форматов
- [ ] Таймаут 5 минут, сохранение что успели

## Phase 3 — Публикация и скачивание

- [ ] Экран одобрения — показ превью, кнопки "Опубликовать" / "Скачать"
- [ ] Puppeteer экспорт PDF (для печатных форматов)
- [ ] Puppeteer экспорт PNG высокого разрешения (для цифровых)
- [ ] Запись в activity_feed при публикации

## Phase 4 — Галерея

- [ ] Общая галерея с фильтрами
- [ ] "Взять за основу" — передача референса в форму
- [ ] Лента активности в реальном времени
- [ ] Мои макеты

## Phase 5 — Администрирование

- [ ] Invite flow (email через Resend)
- [ ] Страница управления командой для admin
- [ ] Суперадмин: список workspace, лимиты, ручное начисление

## Phase 6 — Лимиты и аналитика

- [ ] Лимиты (атомарная проверка в Postgres)
- [ ] Отображение использования в admin панели
- [ ] Автоочистка черновиков через pg_cron

---

## Бэклог

- Чат-интерфейс для настройки брендбука (мастерадмин)
- Браузерные push-уведомления когда генерация готова
- Дашборд потребления с графиками
- Уведомление когда workspace приближается к лимиту
- Мульти-workspace для одного пользователя
- Настройка приватности activity feed
- Fun статусы генерации
- Перегенерация отдельных форматов без пересоздания батча
- Поддержка дополнительных языков интерфейса
